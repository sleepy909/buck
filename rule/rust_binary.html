rust_binary()

This is liable to change in the future.

A rust_binary() rule builds a native executable from the supplied set of Rust source files and dependencies.

Note: Buck is currently tested with (and therefore supports) version 1.1.0 of Rust.

Arguments

  • name (required) The name of the rule.
  • srcs (required)

    The set of Rust source files to be compiled by this rule.

    One of these files must be named {name}.rs or main.rs. This file will be passed to rustc as the crate root.

  • deps (defaults to []) The set of dependencies of this rule. Currently, this only supports rust_library rules.
  • features (defaults to [])

    The set of features to be enabled for this rule.

    These are passed to rustc with --cfg feature="{feature}", and can be used in the code with #[cfg(feature = "{feature}")].

  • rustc_flags (defaults to []) The set of additional compiler flags to pass to rustc.

Examples

For more examples, check out our integration tests.

rust_binary(
  name='greet',
  srcs=[
    'greet.rs',
  ],
  deps=[
    ':greeting',
  ],
)

rust_library(
  name='greeting',
  srcs=[
    'greeting.rs',
  ],
  deps=[
    ':join',
  ],
)

rust_library(
  name='join',
  srcs=[
    'join.rs',
  ],
)
